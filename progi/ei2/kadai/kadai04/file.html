part A

  1. データの性質を表す適切な型名を決める
  2. 1.で決めた型を扱う操作関数の洗い出しを行う
    (最低限、アクセサー、コンストラクターは含まれる)

    3. part B

  3. 1.のデータ型のデータ構造を考える
  4. 2.の操作関数郡の外部仕様を決める
  5. 2.の操作関数郡を実装する

    6. part C

  6. 4.の操作関数群を用いて目的のアプリケーションを作る

A, B, Cは独立して行える...大規模なプログラムでは異なる人やチームで行うのが一般的

  • 1, 4 -> 抽象データ型ん外部仕様の策定
  • 3, 5 -> 抽象データ型の内部仕様の実装

留意事項

  • 2, 3の作業は、使う人の立場に立って操作関数を設計する
    • 同じ処理でも必要なら使い方の異る複数の関数を用意する
    • 関数名は処理内容がわかる適切な名前をつける
  • 操作関数を作る時は、極力データ構造に依存しないようにする。
      データ構造に依存するのは、アクセサーやコンストラクターのみが理想
